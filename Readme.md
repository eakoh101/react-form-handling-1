 value={message} เป็นเรื่องของ Syntax (ไวยากรณ์) ของ JSX ครับ
เหตุผลที่ต้องใส่ { } มีดังนี้ครับ:

* เพื่อบอกว่านี่คือ "ตัวแปร" (JavaScript Expression): ใน JSX ถ้าคุณเขียน value="message" (ใช้เครื่องหมายคำพูด) React จะมองว่าค่าในช่อง Input คือตัวอักษรคำว่า "message" ตรงๆ ตลอดไป
* การดึงค่าจาก State: การใส่ {message} เป็นการบอก React ว่า "ไปเอาค่าที่อยู่ในตัวแปร message มาแสดงผลในช่องนี้ทีนะ"

## สรุปสั้นๆ:

* { } ใน HTML ของ React (JSX) = "เฮ้ React! ตรงนี้ไม่ใช่ตัวอักษรธรรมดานะ ไปอ่านค่าจาก JavaScript มาใส่ให้หน่อย"
* มันไม่ใช่การสร้าง Object เหมือนใน setGreeting({message}) ที่คุณเจอ Error ก่อนหน้านี้ครับ (อันนั้นอยู่ในส่วนของ Logic)

ข้อควรจำ:

* ใน Attribute (เช่น value, id, className): { } ใช้เพื่อใส่ตัวแปร
* ใน Function (เช่น setGreeting(...)): ถ้าใส่ { } จะเป็นการสร้าง Object ใหม่ทันทีครับ

อยากให้ลองเปรียบเทียบการใช้ { } ในจุดอื่นๆ ของ Component นี้เพิ่มเติมไหมครับ?
